# 数据比对工具 (DataCompare) - 项目参数文档

> 项目概述：Excel vs PDF 年度报告数值比对工具。用户上传 Excel 指标表和 PDF 报告文件，系统自动从 PDF 中提取数值并与 Excel 目标值进行比对，生成核对报告。

#### 缓存结构
```python
_cache: dict[str, tuple[list[PageText], list[ExtractedNumber], str]]
# key: PDF 文件名
# value: (pages, numbers, full_text)
```

#### source_file 解析规则
1. **URL 模式**：`https?://[^\s;；，,\n]+`
2. **+P页码 模式**：`([^;；\n]+?)\+P(\d+)`
   - 先用分隔符分割，再对每段提取 +P页码
   - core_name 包含"年鉴" → yearbook，否则 → report
3. **去重**：同一类型、同一 core_name、同一 page 的只保留一个

#### PDF 文件名匹配策略（优先级）
1. 精确匹配
2. 包含匹配：core_name 是 pdf_name 的子串（去掉 _ocr/_trans 后缀后）
3. 前缀匹配：core_name 的前 20 个字符匹配
4. 年份匹配：core_name 中的第一个年份在 pdf_name 中

### 2.8 `backend/fuzzy_matcher.py` - 模糊匹配模块

#### MatchScore 数据类
| 字段 | 类型 | 说明 |
|------|------|------|
| `score` | `float` | 0-100 的匹配分数 |
| `is_exact` | `bool` | 是否精确包含匹配 |

#### FuzzyMatcher 类

| 方法 | 参数 | 返回 | 说明 |
|------|------|------|------|
| `__init__(threshold=60)` | `threshold: int` | - | 初始化模糊匹配器 |
| `match(indicator_name, context, aliases)` | `indicator_name: str`, `context: str`, `aliases: list[str]\|None` | `MatchScore` | 将指标名称与上下文文本进行匹配 |

#### 匹配策略（优先级从高到低）
1. **精确包含**：指标名称完整出现在上下文中 → 100 分
2. **别名精确包含**：指标的某个别名完整出现在上下文中 → 95 分
3. **模糊匹配**：使用 `rapidfuzz.fuzz.partial_ratio` 计算相似度 → 实际分数

## 五、核心业务流程

### 5.1 上传流程
```
用户选择 Excel → 点击上传
  → POST /api/upload
  → 保存 Excel 到 uploads/{user_id}/{task_id}/
  → read_indicators() 解析 Excel（自动检测 simple/wide 格式）
  → 提取单元格背景颜色（win32com）
  → 收集所有引用的 PDF 文件名
  → 返回指标列表、颜色列表、匹配的 PDF 列表
```

### 5.2 分析流程
```
用户选择颜色映射 → 点击开始分析
  → POST /api/analyze
  → 按颜色过滤指标
  → 应用 color_mapping 将 source_file 分配到三个互斥字段
  → Comparator.preprocess_pdf() 预处理匹配的 PDF
  → Comparator.compare_indicator() 对每个指标：
    1. 确定来源类型（yearbook/report/url）
    2. _parse_source_file() 解析来源文本
    3. _match_pdf_name() 匹配实际 PDF 文件名
    4. _is_year_match() 年份过滤
    5. _find_matches_on_page() 在指定页码搜索数值
    6. 模糊匹配验证 + 置信度计算
    7. 自动判断状态（已确认/存疑）
  → 返回 AnalysisResponse
```

### 5.3 核对流程
```
用户查看指标 → 右侧显示 PDF 对应页面
  → 确认（Enter）/ 存疑（Space）
  → PUT /api/indicator/{id}/status
  → 自动跳转到下一个未核对指标
  → 底部状态栏实时更新进度
```

### 5.4 导出流程
```
下载报告表 → GET /api/export → 生成 Excel（汇总表+各PDF详情+未找到项）
下载标色原表 → GET /api/export_original → 在原表上标色
下载纯文本 → GET /api/export_text → 提取的 PDF 纯文本
```

## 六、关键设计决策

### 6.1 颜色→来源类型映射
- Excel 单元格背景色用于区分数据类型
- 用户在上传后选择每种颜色对应的来源类型（年鉴/司局/AI）
- 分析时根据映射将 `source_file` 拆分到三个互斥字段

### 6.2 PDF 文件名匹配
- 支持 `_ocr.pdf` / `_trans.pdf` 后缀变体
- 支持年鉴 PDF 的年份匹配
- 支持前缀匹配和包含匹配

### 6.3 数值匹配策略
1. 在指定页码中搜索与 Excel 目标值相等的数值
2. 对每个值匹配做模糊匹配验证（指标名称 vs 上下文）
3. 去重：80 字符内视为同一处
4. 置信度 = 70（值匹配基线）+ 0.3 × 模糊分数

### 6.4 单位换算
- 支持中文单位（万元、亿元、百万元、千元、元、%）
- 通过 `UNIT_MULTIPLIERS` 映射进行换算比较

### 6.5 缓存机制
- 内存缓存：`Comparator._cache` 字典
- 磁盘缓存：`.pdf_cache/` 目录下的 JSON 文件
- 缓存过期检查：比较 PDF 文件修改时间
