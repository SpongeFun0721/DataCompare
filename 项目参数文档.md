# 数据比对工具 (DataCompare) - 项目参数文档

> 项目概述：Excel vs PDF 年度报告数值比对工具。用户上传 Excel 指标表和 PDF 报告文件，系统自动从 PDF 中提取数值并与 Excel 目标值进行比对，生成核对报告。

### 2.5 `backend/comparator.py` - 核心比对引擎

#### Comparator 类

| 方法 | 参数 | 返回 | 说明 |
|------|------|------|------|

| `_highlight_context(context, indicator_name, raw_num, aliases)` | 上下文参数 | `str` | 生成高亮后的上下文 HTML |
| `run_analysis_on_preprocessed(indicators, pdf_names, yearbook_index)` | 指标+PDF名+年鉴索引 | `AnalysisResponse` | 对已预处理的 PDF 执行比对分析 |
| `run_full_analysis(indicators, pdf_dir, yearbook_index)` | 指标+PDF目录+年鉴索引 | `AnalysisResponse` | 执行完整的比对分析（含预处理） |
| `_calc_progress(indicators)` | `indicators: list[Indicator]` | `ProgressInfo` | **静态方法** - 计算核对进度 |

#### 缓存结构
```python
_cache: dict[str, tuple[list[PageText], list[ExtractedNumber], str]]
# key: PDF 文件名
# value: (pages, numbers, full_text)
```

#### source_file 解析规则
1. **URL 模式**：`https?://[^\s;；，,\n]+`，同时提取前面的年份标签（如 `2022年：`）
2. **+P页码 模式**：`([^;；\n]+?)\+P(\d+)`
   - 先用分隔符分割，再对每段提取 +P页码
   - core_name 包含"年鉴" → yearbook，否则 → report
   - **年份标签前缀分离**：年鉴来源自动将 `"2021年：中国体育年鉴2022"` 拆分为 `year_label="2021年"`、`core_name="中国体育年鉴2022"`
3. **去重**：同一类型、同一 core_name、同一 page 的只保留一个

#### PDF 文件名匹配策略（综合打分）
1. **机构名匹配**（权重 40）：提取"协会/中心/司/大学"等核心机构名
2. **年份对匹配**（权重 50）：提取 PDF 中的两个连续年份进行匹配
3. **归一化文本相似度**（权重 30）：去忽略词后计算字符交集比例
4. 总分 >= 30 视为有效匹配

#### 年鉴 PDF 匹配（特判）
- 不按年份严格过滤（2022 年年鉴也可能包含 2021 年数据）
- 优先匹配 core_name 中的年鉴出版年份，再匹配指标年份，最后使用默认年鉴

   - `yearbook`/`report` → 使用 `matched_pdf_name` 和 `matched_page` 打开 PDF
3. 高亮文本：有匹配用匹配值，无匹配用 `target_value`

## 五、核心业务流程

### 5.1 上传流程
```
用户选择 Excel → 点击上传
  → POST /api/upload
  → 保存 Excel 到 uploads/{user_id}/
  → read_indicators() 解析 Excel（自动检测 simple/wide 格式）
  → 提取单元格背景颜色（win32com）
  → 在 shared_pdfs 中匹配所有引用的 PDF 文件名
  → 返回指标列表、颜色列表、匹配的 PDF 列表
```

### 5.2 分析流程
```
用户选择颜色并确认映射 → 点击开始分析
  → POST /api/analyze
  → 按 selected_colors 过滤指标
  → 应用 color_mapping 将 source_file 分配到三个互斥字段
  → Comparator.preprocess_pdf() 预处理匹配的 PDF（含磁盘缓存）
  → Comparator.compare_indicator() 对每个指标：
    1. 确定来源类型（yearbook/report/url）
    2. _parse_source_file() 解析来源文本（分离年份标签前缀）
    3. 年鉴→yearbook_index 按年份匹配 / 司局→_match_pdf_name() 综合打分匹配
    4. _is_year_match() 司局报告年份过滤（年鉴特判不过滤）
    5. _find_matches_on_page() 在指定页及前后一页搜索数值
    6. 模糊匹配验证 + 置信度计算
    7. 自动判断状态：唯一高置信度匹配→已核对，否则→未核对
  → 返回 AnalysisResponse（indicators + pdf_names + results + progress）
```

### 5.3 核对流程
```
用户查看指标 → 右侧显示 PDF 对应页面（或 URL 网页文本）
  → 确认（Enter）/ 未核对（Space）
  → PUT /api/indicator/{id}/status
  → 自动跳转到下一个未核对指标
  → 底部状态栏实时更新进度
  → 支持批注（自动保存到上一个指标）
```

### 5.4 AI 分析流程
```
当指标来源类型为 url 时：
  → SSE /api/analyze-stream 流式获取多个 URL 网页文本
  → 用户点击"AI 分析"
  → POST /api/ai-analyze
  → DeepSeek API 判断数据是否存在
  → 返回匹配值、运算逻辑、分析说明
  → 支持刷新缓存（POST /api/refresh-cache）
```

### 5.5 导出流程
```
下载报告表 → GET /api/export → 生成 Excel（汇总表+各PDF详情+未找到项）
下载标色原表 → GET /api/export_original → 在原表上按核对状态标色
下载纯文本 → GET /api/export_text → 提取的 PDF 纯文本
```

## 六、关键设计决策

### 6.1 颜色→来源类型映射
- Excel 单元格背景色用于区分数据类型（绿色→年鉴、蓝色→司局、红色→AI）
- 用户在上传后可在 UI 中手动覆盖每种颜色的来源类型
- 分析时根据映射将 `source_file` 拆分到 `source_file_yearbook` / `source_file_report` / `source_file_url` 三个互斥字段

### 6.2 PDF 文件名匹配
- 支持 `_ocr.pdf` / `_trans.pdf` 后缀变体
- **司局报告**：提取机构名 + 年份对 + 归一化文本综合打分（阈值 30 分）
- **年鉴**：按年份在 `yearbook_index` 中匹配，不按年份过滤

### 6.3 数值匹配策略
1. 在指定页及前后一页搜索与 Excel 目标值相等的数值
2. 对每个值匹配做模糊匹配验证（指标名称 vs 上下文）
3. 去重：80 字符内视为同一处
4. 置信度 = 70（值匹配基线）+ 0.3 × 模糊分数

### 6.4 单位换算
- 支持中文单位（万元、亿元、百万元、千元、元、%）
- 通过 `UNIT_MULTIPLIERS` 映射进行换算比较

### 6.5 缓存机制
- **内存缓存**：`Comparator._cache` 字典
- **磁盘缓存**：`.pdf_cache/` 目录下的 JSON 文件，缓存过期检查 PDF 修改时间
- **IndexedDB**：前端预加载司局 PDF 并缓存到浏览器本地
- **网页缓存**：URL 网页文本通过 SSE 流式获取后缓存，支持手动刷新

### 6.6 多用户隔离
- 每次上传创建独立的数据字典，通过 `X-User-Id` 请求头隔离
- 上传文件存放在 `uploads/{user_id}/` 目录

### 6.7 域名限流
- 全局共享 `DomainRateLimiter`，防止高并发请求击垮目标服务器
- 每域名独立配置速率（默认 2 req/s，特定域名 1.5 req/s）
- 用户级 `UserTaskManager` 管理请求队列、重试、超时

### 6.8 自动确认策略
自动标记"已核对"条件（需同时满足）：
1. 来源非混合（非 report+yearbook+url 并存）
2. 非多来源数值（如"年报:200 报告:300"）
3. 全局仅 1 次完全匹配
4. 置信度 >= 70%
